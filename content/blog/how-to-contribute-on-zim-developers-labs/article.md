## 1. Bug Squad Guide

If you just hate seeing things that don't work then you are on the right guide. In this guide are steps on how to contribute to Zim Developers Labs by finding and fixing bugs

1. **Find a bug**: You might just bump into a bug on one of the Zim Developers Labs projects or you can activily go digging for bugs.
2. **Report the bug**: Once you see a bug create an issue on the project's github issues section. (Private repos are handled differently [read the conversation](#)).
3. **Fix the bug**: Before you fix a bug wait to be assigned to the issue by one of the project managers then proceed to follow the [code contributions guide](#).

### Bug Squad Contribution Impact Points

- For finding and reporting a bug you generally get 25 impact points
- If your rank gives you access to the project source code proceeding to fix the bug will give you x impact points

## 2. English Teachers Guide

Yes you, I know you like dictionary grade grammar. If I'm right then follow this guide. I will go through steps on how to contribute to Zim Developers Labs by finding and correcting grammar errors.

1. **Spot the error**: If you find a typo or some bad grammar on any of the community projects proceed to actually confirm if its bad grammar or just a statement you dont prefer which in that case continue scrolling 😊.
2. **Create issue**: Create issue on the related github repo on the project's github issues section.
3. **Fix the error**: You don't need to create a separate branch for typo and grammar fixes you can create a pull request on the main branch.

### English Teacher Contribution Impact Points

- For spotting and creating a grammar or typo issue you generally get 25 impact points
- If your ranks gives you access to the project source code proceed to fix a typo or grammar issue will give you x impact points
